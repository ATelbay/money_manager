# Git Conventional Commits

## Process

1. Выполни `git status` и `git diff`, чтобы проанализировать незакоммиченные изменения.
2. Сформируй сообщение для коммита строго в формате: `<type>(<scope>): <subject>`
3. Выбери правильный `<type>`:
   - `feat`: новая функциональность (например, парсер PDF).
   - `fix`: исправление бага.
   - `test`: добавление или обновление UI-тестов (очень важно для данного проекта!).
   - `refactor`: рефакторинг кода без изменения логики.
   - `chore`: обновление зависимостей (в `libs.versions.toml`), настройки сборки.
4. В качестве `<scope>` укажи затронутый Gradle-модуль или фичу (например, `import`, `database`, `ui`).
5. В теле коммита (body) кратко объясни *причину* изменений (почему это было сделано), а не просто перечисляй измененные файлы.
6. Запроси подтверждение перед выполнением `git commit`.
