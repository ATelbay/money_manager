## Codebase Patterns
- Convention Plugins in build-logic/convention/ for shared build config
- Type-Safe Navigation for routes in app module
- Room entities live in core:database, domain models in core:model
- ExchangeRate domain model now uses `quotes: Map<String, Double>` (code → KZT per 1 unit) with KZT as implicit pivot (rate 1.0)
- ExchangeRateRepository uses `observeQuotes()`, `saveQuotes()`, `fetchAndStoreQuotes()` method names
- ConvertAmountUseCase accepts `sourceCurrency`, `targetCurrency`, `quotes` (not direction enum)
- ObserveExchangeRateUseCase replaces GetUsdKztRateUseCase
- UserPreferences stores full quotes as JSON in `exchange_quotes_json` key; legacy `usd_kzt_rate` key kept for backward compat
- StoredExchangeRate has both `usdToKzt: Double` and `quotes: Map<String, Double>?` — mapper uses `quotes ?: mapOf("USD" to usdToKzt)`
- NbkExchangeRateRemoteDataSource.parseAllQuotes() extracts all currency rows; normalizes nominal (divides by 10/100)
- UserPreferences.quotesToJson/jsonToQuotes: simple hand-rolled JSON serializer (no Gson/org.json dependency)
- Data layer ExchangeRateCacheModel and NbkExchangeRateRemoteModel use `quotes: Map<String, Double>`
- TransactionListViewModel.convertToBaseCurrency() catches IllegalArgumentException from ConvertAmountUseCase for unsupported pairs
- SettingsViewModel uses `combine(baseCurrency, targetCurrency, observeExchangeRateUseCase())` pattern for atomic state updates
- Main added CurrencyPickerRoute/CurrencyPickerScreen for currency selection via navigation (not bottom sheet)
- UserPreferences has `quoteRefreshFailureCount` Flow and `incrementQuoteRefreshFailureCount()`/`resetQuoteRefreshFailureCount()` for tracking consecutive failures
- TransactionDao.getCurrencyTransactionCounts() joins transactions+accounts for currency frequency; returns CurrencyTransactionCount POJO
- TransactionRepository.getTopCurrenciesByUsage() returns currency codes ranked by all-time transaction count (ties broken alphabetically)
- SettingsViewModel auto-switches base/target pair after 3 consecutive refresh failures using transaction frequency data
- presentation:settings depends on domain:transactions for accessing TransactionRepository
- Full fallback policy: `canConvertAll` flag computed once from period scope, shared by mapTransactionRows() and resolveSummaryMetrics() — prevents partial mixed conversion
