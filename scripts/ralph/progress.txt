## Codebase Patterns
- New cross-cutting business logic belongs in its own `:domain:*` module and must be registered in `settings.gradle.kts`.
- Domain modules can apply `moneymanager.android.hilt` to support `@Inject` use cases even when they contain only Kotlin business logic.
- Small cross-feature cached settings or lookup values can live in `core:datastore` `UserPreferences`, with data modules mapping the stored preference model into domain models.
- `presentation:settings` stores persisted options in `UserPreferences` as simple string codes, then maps them into typed enums inside `SettingsViewModel` for UI state.
- When a screen shows converted summary totals but unconverted row amounts, keep a separate `displayCurrency` in UI state instead of reusing the per-row `currency` label.
- Feature ViewModel unit tests in this repo can stay as pure JVM tests by wiring fake repositories plus mocked `UserPreferences` flows and driving coroutine timing with `StandardTestDispatcher`.
