# Ralph Progress Log
## Codebase Patterns
- In `core:ui`, shared cross-screen components should read semantic colors from `MoneyManagerTheme.colors` instead of hardcoding palette constants, so light/dark variants can diverge safely in one place.
- In `presentation:settings`, persist lightweight selector state in `UserPreferences` as raw string codes and map those codes to richer UI labels through a feature-local catalog.
- Long in-place Compose pickers should cap `DropdownMenu` height to avoid overflow and clipped popups on smaller devices.
- When multiple persisted selectors share one catalog, resolve unknown codes with a field-specific fallback instead of a single global default.
- In `presentation:settings`, when an option set grows beyond a compact inline control, prefer a `TopAppBar` action that opens a `ModalBottomSheet` with search, and keep the screen body to a concise summary row.
- Compose layout helpers such as `weight` and `matchParentSize()` come from the active `Row`/`Column`/`Box` scope in this codebase; after UI refactors, verify both the scope and the import block before assuming unresolved references are missing imports.
- In this Compose setup, `Modifier.weight(...)` and `matchParentSize()` should be used from the surrounding `Row`/`Column`/`Box` scope; importing them explicitly can bind the wrong symbol and break compilation.
- For top-level screens under the shared bottom navigation, avoid stacking an inner `Scaffold`'s bottom content padding with a manual trailing spacer; use explicit list `contentPadding` for the FAB clearance instead.
- For top-level `Scaffold` screens with a FAB, prefer `LazyColumn` bottom padding derived from `padding.calculateBottomPadding()` plus a small gutter instead of hardcoding the FAB clearance; it tracks insets while keeping spacing aligned with neighboring screens.
- In the app-level `Scaffold`, keep the shared bottom bar mounted and animate its visibility instead of removing the slot immediately on route changes; that lets the bottom inset collapse smoothly and avoids abrupt FAB repositioning on the outgoing top-level screen.
