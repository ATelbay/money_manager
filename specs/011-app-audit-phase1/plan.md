# Implementation Plan: App Audit Phase 1

**Branch**: `011-app-audit-phase1` | **Date**: 2026-03-20 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/011-app-audit-phase1/spec.md`

## Summary

Fix 5 critical/high-priority issues from a full-app code review: (1) exclude financial database from cloud backup, (2) fix stuck save buttons in Account/Category edit ViewModels, (3) extract ~22 hardcoded Russian strings into the AppStrings localization system, (4) add missing database indexes with Room migration 3→4, and (5) extract SyncManager behind a domain interface to fix a presentation→data architecture violation.

## Technical Context

**Language/Version**: Kotlin 2.3.0
**Primary Dependencies**: Jetpack Compose (BOM 2026.01.01), Hilt 2.58, Room 2.8.4, Navigation Compose 2.9.7
**Storage**: Room (SQLite) — `money_manager.db`, Preferences DataStore
**Testing**: JUnit 4, MockK, Turbine, kotlinx-coroutines-test
**Target Platform**: Android (minSdk defined in convention plugins)
**Project Type**: Mobile app (Android)
**Performance Goals**: Index-backed queries for statistics (no full table scans on 1000+ transactions)
**Constraints**: Backwards-compatible Room migration, offline-first, no destructive migrations
**Scale/Scope**: ~36 Gradle modules, 3 entities, 5 audit items

## Implementation Order

Tasks should be implemented in this dependency order:

1. **S1 — Backup rules** (standalone, no deps)
2. **D1 — Database indexes + migration** (standalone, no deps)
3. **A1 — Sync domain interface** (standalone, enables A2/L1 for SettingsViewModel)
4. **A2 — Save button error handling** (can overlap with L1)
5. **L1+L2 — Localization strings** (depends on A2 for ViewModel error pattern; A1 for SettingsViewModel)

Items 1-3 can be done in parallel. Items 4-5 should follow after A1 is merged.
